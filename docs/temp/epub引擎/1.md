下面给你一套**可以直接落地的 EPUB 引擎实现代码（基于 Readium Kotlin Toolkit 3.x）**，并且严格遵循你前面定的模块边界：`feature/*` 只面向 `core/reader/api + core/reader/runtime`，**不直接依赖 engines**；EPUB 引擎放在 `engines/epub`，只实现 `ReaderEngine/ReaderDocument/ReaderSession/ReaderController`。

> 关键点：
>
> * EPUB 的“专业实现”在 Android 上基本必须依赖成熟的排版/定位/选择/高亮/分页体系。**Readium 的 EpubNavigatorFragment 是当前最靠谱的工业级方案**（CFI/Locator、分页、样式偏好、搜索服务、DRM/LCP 扩展）。
> * 因为 Readium 的 Navigator 是 UI 组件（Fragment/WebView），你的 `ReaderController` 必须支持 “**绑定渲染 Surface**” 的能力（否则引擎无法真正渲染/翻页/定位）。
> * 下面代码会对 `core/reader/api` 做一个非常小但关键的增强：`RenderSurface + bindSurface/unbindSurface`，并新增 `RenderContent.Embedded` 表示“渲染由引擎在 Surface 内部完成”。

```kotlin
import android.content.Context
import org.readium.r2.shared.publication.ContentProtection
import org.readium.r2.shared.util.http.DefaultHttpClient
import org.readium.r2.streamer.PublicationOpener

/**
 * 把 Readium 的重对象集中创建，避免每次 open 都 new 一堆。
 *
 * - DefaultHttpClient/AssetRetriever/PublicationParser/PublicationOpener 都适合长生命周期复用
 * - ContentProtections 用于 DRM/LCP（可选注入）

    contentProtections: List<ContentProtection> = emptyList(),

    val httpClient = DefaultHttpClient()
```

## C3) 打开/解析：EpubOpener.kt

**File:** `engines/epub/src/main/kotlin/com/example/reader/engines/epub/internal/open/EpubOpener.kt`

```kotlin
package com.example.reader.engines.epub.internal.open

import android.content.Context
import com.example.reader.api.error.ReaderError
import com.example.reader.api.error.ReaderResult
import com.example.reader.api.open.OpenOptions
import com.example.reader.engines.epub.internal.readium.ReadiumEpubToolkit
import com.example.reader.model.BookFormat
import com.example.reader.model.DocumentCapabilities
import com.example.reader.model.DocumentId
import com.example.reader.source.DocumentSource
import java.util.UUID
import kotlinx.coroutines.CoroutineDispatcher
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.withContext
import org.readium.r2.shared.publication.Publication
import org.readium.r2.shared.util.Try
import org.readium.r2.shared.util.mediatype.MediaType
import org.readium.r2.shared.util.toAbsoluteUrl

internal class EpubOpener(
    context: Context,
    private val ioDispatcher: CoroutineDispatcher = Dispatchers.IO,
    private val toolkit: ReadiumEpubToolkit = ReadiumEpubToolkit(context)
) {
    private val appContext = context.applicationContext

    suspend fun open(
        source: DocumentSource,
        options: OpenOptions
    ): ReaderResult<EpubDocument> = withContext(ioDispatcher) {
        try {
            val absUrl = source.uri.toAbsoluteUrl()
                ?: return@withContext ReaderResult.Err(
                    ReaderError.CorruptOrInvalid("Uri cannot be converted to AbsoluteUrl: ${source.uri}")
                )

            val hintedMediaType = source.mimeType?.let { runCatching { MediaType(it) }.getOrNull() }
            val mediaType = hintedMediaType ?: MediaType.EPUB

            val asset = toolkit.assetRetriever
                .retrieve(absUrl, mediaType)
                .getOrElse { err ->
                    return@withContext ReaderResult.Err(
                        ReaderError.Io("Failed to retrieve asset: $err")
                    )
                }

            val publication: Publication = toolkit.publicationOpener
                .open(asset, allowUserInteraction = true)
                .getOrElse { openError ->
                    // DRM/损坏/格式错误都在这里
                    return@withContext ReaderResult.Err(
                        ReaderError.CorruptOrInvalid("Failed to open publication: $openError")
                    )
                }

            if (publication.isRestricted) {
                // Readium 表示资源访问受限，Navigator 不能渲染
                runCatching { publication.close() }
                runCatching { asset.close() }
                return@withContext ReaderResult.Err(
                    ReaderError.DrmRestricted("Publication is restricted and can't be rendered")
                )
            }

            val docId = DocumentId(
                publication.metadata.identifier
                    ?: stableIdFromUri(source.uri.toString())
            )

            val caps = DocumentCapabilities(
                reflowable = true,       // Readium 支持 reflow/FXL，但这里先用统一能力表达
                fixedLayout = false,     // 若你要精确区分 FXL，可按 publication.metadata.presentation 再细化
                outline = publication.tableOfContents.isNotEmpty(),
                search = publication.isSearchable,
                textExtraction = false,  // 你要做复制/导出可再接 ContentService，这里先关闭能力避免“假支持”
                annotations = true,      // 先声明支持（高亮装饰基于 Navigator 可做），你的 Repo/映射可后续接入
                links = true
            )

            ReaderResult.Ok(
                EpubDocument(
                    id = docId,
                    format = BookFormat.EPUB,
                    capabilities = caps,
                    openOptions = options,
                    publication = publication,
                    asset = asset
                )
            )
        } catch (t: Throwable) {
            ReaderResult.Err(mapThrowable(t))
        }
    }

    private fun stableIdFromUri(uri: String): String =
        UUID.nameUUIDFromBytes(uri.toByteArray()).toString()

    private fun mapThrowable(t: Throwable): ReaderError =
        when (t) {
            is ReaderError -> t
            is java.io.FileNotFoundException -> ReaderError.NotFound(cause = t)
            is SecurityException -> ReaderError.PermissionDenied(cause = t)
            is java.util.zip.ZipException -> ReaderError.CorruptOrInvalid(cause = t)
            is java.io.IOException -> ReaderError.Io(cause = t)
            else -> ReaderError.Internal(cause = t)
        }
}
```

## C6) Locator 映射：用 Readium Locator JSON 做你自己的 Locator（稳定/专业）

因为 Readium 官方 Locator 是完整定位模型（href + mediaType + fragments + totalProgression 等），**建议你的 `core/model/Locator` 对 EPUB 用 JSON 承载 Readium Locator**，而不是只存 CFI。这样：

* 书签/进度恢复/跨版本兼容更好
* 搜索结果/TOC 跳转更直接
* 不会卡在“CFI 单独无法映射到 href”的坑里

**File:** `engines/epub/src/main/kotlin/com/example/reader/engines/epub/internal/locator/ReadiumLocatorMapper.kt`

```kotlin
package com.example.reader.engines.epub.internal.locator

import com.example.reader.model.Locator
import org.json.JSONObject
import org.readium.r2.shared.publication.Locator as ReadiumLocator

internal object ReadiumLocatorSchemes {
    const val READIUM_LOCATOR_JSON = "readium.locator.json"
}

internal fun ReadiumLocator.toAppLocator(): Locator =
    Locator(
        scheme = ReadiumLocatorSchemes.READIUM_LOCATOR_JSON,
        value = this.toJSON().toString(),
        extras = buildMap {
            put("href", this@toAppLocator.href.toString())
            put("mediaType", this@toAppLocator.mediaType.toString())
            this@toAppLocator.locations.totalProgression?.let { put("totalProgression", it.toString()) }
            this@toAppLocator.locations.position?.let { put("position", it.toString()) }
            this@toAppLocator.title?.let { put("title", it) }
        }
    )

internal fun Locator.toReadiumLocatorOrNull(): ReadiumLocator? =
    when (scheme) {
        ReadiumLocatorSchemes.READIUM_LOCATOR_JSON -> runCatching {
            ReadiumLocator.fromJSON(JSONObject(value))
        }.getOrNull()

        else -> null
    }
```

```kotlin
import com.example.reader.model.Progression
import java.io.Closeable
import java.util.concurrent.atomic.AtomicReference
import kotlinx.coroutines.CoroutineDispatcher
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.SupervisorJob
import kotlinx.coroutines.cancel
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.MutableSharedFlow
import kotlinx.coroutines.flow.MutableStateFlow
import kotlinx.coroutines.flow.StateFlow
import kotlinx.coroutines.flow.asSharedFlow
import kotlinx.coroutines.flow.filter
import kotlinx.coroutines.flow.first
import kotlinx.coroutines.launch
import kotlinx.coroutines.sync.Mutex
import kotlinx.coroutines.sync.withLock

import kotlinx.coroutines.withTimeoutOrNull
import org.readium.r2.navigator.epub.EpubNavigatorFactory
import org.readium.r2.navigator.epub.EpubNavigatorFragment
import org.readium.r2.shared.publication.Publication
import org.readium.r2.shared.publication.Locator as ReadiumLocator

internal class EpubController(
    private val publication: Publication,
    initialLocator: Locator?,
    initialConfig: RenderConfig,
    private val mainDispatcher: CoroutineDispatcher = Dispatchers.Main
) : ReaderController {

    private val scope = CoroutineScope(SupervisorJob() + mainDispatcher)
    private val navMutex = Mutex()

    private val _events = MutableSharedFlow<ReaderEvent>(extraBufferCapacity = 64)
    override val events: Flow<ReaderEvent> = _events.asSharedFlow()

    private val _state = MutableStateFlow(
        RenderState(
            locator = initialLocator ?: Locator("init", "0"),
            progression = Progression(percent = 0.0, label = null),
            nav = NavigationAvailability(canGoPrev = true, canGoNext = true),
            titleInView = null,
            config = initialConfig
        )
    )
    override val state: StateFlow<RenderState> = _state

    private var constraints: LayoutConstraints? = null

    // bindSurface 后才有 fragment
    private val fragmentRef = AtomicReference<EpubNavigatorFragment?>(null)
    private var surface: FragmentRenderSurface? = null
    private var locatorCollector: Closeable? = null

    // 在未绑定 surface 前保存 pending locator
    private var pendingLocator: Locator? = initialLocator

    override suspend fun bindSurface(surface: RenderSurface): ReaderResult<Unit> {
        val fragSurface = surface as? FragmentRenderSurface
            ?: return ReaderResult.Err(ReaderError.Internal("EPUB engine requires FragmentRenderSurface"))

        return withContext(mainDispatcher) {
            try {
                this@EpubController.surface = fragSurface

                val fm = fragSurface.fragmentManager
                if (fm.isStateSaved) {
                    return@withContext ReaderResult.Err(
                        ReaderError.Internal("FragmentManager state is saved; cannot attach navigator now.")
                    )
                }

                val tag = buildFragmentTag()
                val existing = fm.findFragmentByTag(tag)
                val fragment = when (existing) {
                    is EpubNavigatorFragment -> existing
                    else -> createAndAttachFragment(fm, fragSurface.containerViewId, tag)
                }

                fragmentRef.set(fragment)

                // 应用 config -> Readium preferences
                applyConfigToNavigator(fragment, _state.value.config)

                // 绑定后如果有 pending locator，跳转过去
                pendingLocator?.toReadiumLocatorOrNull()?.let { rl ->
                    fragment.go(rl, animated = false)
                    pendingLocator = null
                }

                // 监听 currentLocator -> 更新 state/events
                startCollectingLocator(fragment)

                ReaderResult.Ok(Unit)
            } catch (t: Throwable) {
                ReaderResult.Err(mapThrowable(t))
            }
        }
    }

    override suspend fun unbindSurface(): ReaderResult<Unit> {
        // 专业实现建议：UI 短暂重组时不要移除 fragment（避免重排/闪烁）。
        // 这里做“解除引用 + 停止收集”，真正销毁在 close() 完成。
        return withContext(mainDispatcher) {
            try {
                locatorCollector?.close()
                locatorCollector = null
                this@EpubController.surface = null
                ReaderResult.Ok(Unit)
            } catch (t: Throwable) {
                ReaderResult.Err(mapThrowable(t))
            }
        }
    }

    override suspend fun setLayoutConstraints(constraints: LayoutConstraints): ReaderResult<Unit> {
        // Readium Navigator 内部根据 WebView/Viewport 自行处理，这里主要存档即可（用于你后续扩展策略）
        this.constraints = constraints
        return ReaderResult.Ok(Unit)
    }

    override suspend fun setConfig(config: RenderConfig): ReaderResult<Unit> {
        _state.value = _state.value.copy(config = config)

        val fragment = fragmentRef.get()
        if (fragment != null) {
            return withContext(mainDispatcher) {
                try {
                    applyConfigToNavigator(fragment, config)
                    ReaderResult.Ok(Unit)
                } catch (t: Throwable) {
                    ReaderResult.Err(mapThrowable(t))
                }
            }
        }
        // 未绑定 surface：先缓存，bind 时再应用
        return ReaderResult.Ok(Unit)
    }

    override suspend fun render(policy: RenderPolicy): ReaderResult<RenderPage> {
        val current = _state.value
        return ReaderResult.Ok(
            RenderPage(
                id = PageId("epub:${current.locator.value.hashCode()}"),
                locator = current.locator,
                content = com.example.reader.api.render.RenderContent.Embedded,
                links = emptyList(),
                decorations = emptyList(),
                metrics = null
            )
        )
    }

    override suspend fun next(policy: RenderPolicy): ReaderResult<RenderPage> =
        navMutex.withLock {
            val fragment = fragmentRef.get()
                ?: return ReaderResult.Err(ReaderError.Internal("Surface not bound"))

            val before = _state.value.locator.value
            val moved = withContext(mainDispatcher) { fragment.goForward(animated = true) }
            if (!moved) return render(policy)

            awaitPageChange(before)
        }

    override suspend fun prev(policy: RenderPolicy): ReaderResult<RenderPage> =
        navMutex.withLock {
            val fragment = fragmentRef.get()
                ?: return ReaderResult.Err(ReaderError.Internal("Surface not bound"))

            val before = _state.value.locator.value
            val moved = withContext(mainDispatcher) { fragment.goBackward(animated = true) }
            if (!moved) return render(policy)

            awaitPageChange(before)
        }

    override suspend fun goTo(locator: Locator, policy: RenderPolicy): ReaderResult<RenderPage> =
        navMutex.withLock {
            val fragment = fragmentRef.get()
                ?: run {
                    // 未绑定时先缓存，等 bindSurface 后跳
                    pendingLocator = locator
                    return render(policy)
                }

            val rl = locator.toReadiumLocatorOrNull()
                ?: return ReaderResult.Err(ReaderError.CorruptOrInvalid("Unsupported EPUB locator scheme: ${locator.scheme}"))

            val before = _state.value.locator.value
            val moved = withContext(mainDispatcher) { fragment.go(rl, animated = false) }
            if (!moved) return render(policy)

            awaitPageChange(before)
        }

    override suspend fun goToProgress(percent: Double, policy: RenderPolicy): ReaderResult<RenderPage> =
        navMutex.withLock {
            val fragment = fragmentRef.get()
                ?: return ReaderResult.Err(ReaderError.Internal("Surface not bound"))

            val target: ReadiumLocator = publication.locateProgression(percent)
                ?: return ReaderResult.Err(ReaderError.Internal("Cannot locate progression=$percent"))

            val before = _state.value.locator.value
            val moved = withContext(mainDispatcher) { fragment.go(target, animated = false) }
            if (!moved) return render(policy)

            awaitPageChange(before)
        }

    override suspend fun prefetchNeighbors(count: Int): ReaderResult<Unit> {
        // Readium 内部会做一定缓存/预取；如需强预取可扩展 Publication.get/Resource 缓存
        return ReaderResult.Ok(Unit)
    }

    override suspend fun invalidate(reason: InvalidateReason): ReaderResult<Unit> {
        // Readium 没有直接对外的 invalidate API；你的主题/字体变更走 submitPreferences 即可
        return ReaderResult.Ok(Unit)
    }

    override fun close() {
        runCatching { locatorCollector?.close() }
        locatorCollector = null

        // 尝试移除 fragment（释放 WebView/资源）
        val fragSurface = surface
        val fragment = fragmentRef.getAndSet(null)
        if (fragSurface != null && fragment != null) {
            val fm = fragSurface.fragmentManager
            if (!fm.isStateSaved) {
                runCatching {
                    fm.beginTransaction()
                        .remove(fragment)
                        .commitNowAllowingStateLoss()
                }
            }
        }

        scope.cancel()
    }

    // ----------------- internal helpers -----------------

    private fun buildFragmentTag(): String =
        "ReadiumEpubNavigator:${publication.metadata.identifier ?: publication.metadata.title.toString()}"

    private fun createAndAttachFragment(
        fm: androidx.fragment.app.FragmentManager,
        containerId: Int,
        tag: String
    ): EpubNavigatorFragment {
        val factory = EpubNavigatorFactory(publication)
        val fragmentFactory = factory.createFragmentFactory()
        val fragment = fragmentFactory.instantiate(
            publication::class.java.classLoader!!,
            EpubNavigatorFragment::class.java.name
        ) as EpubNavigatorFragment

        fm.beginTransaction()
            .replace(containerId, fragment, tag)
            .commitNow()

        return fragment
    }

    private fun applyConfigToNavigator(fragment: EpubNavigatorFragment, config: RenderConfig) {
        // 目前只对 ReflowText 映射偏好；FXL 的缩放/旋转通常属于 PDF/图像系引擎策略
        val prefs = config.toEpubPreferences()
        fragment.submitPreferences(prefs)
    }

    private fun startCollectingLocator(fragment: EpubNavigatorFragment) {
        // 避免重复 collect
        locatorCollector?.close()

        val job = scope.launch {
            fragment.currentLocator.collect { rl ->
                val appLocator = rl.toAppLocator()
                val percent = rl.locations.totalProgression ?: 0.0
                val pos = rl.locations.position
                val label = when {
                    pos != null -> "#$pos"
                    else -> null
                }

                _state.value = _state.value.copy(
                    locator = appLocator,
                    progression = Progression(percent = percent, label = label),
                    nav = NavigationAvailability(canGoPrev = true, canGoNext = true),
                    titleInView = rl.title
                )

                _events.tryEmit(ReaderEvent.PageChanged(appLocator))
            }
        }

        locatorCollector = Closeable { job.cancel() }
    }

    private suspend fun awaitPageChange(beforeValue: String): ReaderResult<RenderPage> {
        // 等待 state.locator 变化，避免 goForward 后立即 render 仍是旧状态
        withTimeoutOrNull(2_000) {
            state.filter { it.locator.value != beforeValue }.first()
        }
        return render(RenderPolicy.Default)
    }

    private fun mapThrowable(t: Throwable): ReaderError =
        when (t) {
            is ReaderError -> t
            is SecurityException -> ReaderError.PermissionDenied(cause = t)
            is java.io.FileNotFoundException -> ReaderError.NotFound(cause = t)
            is java.util.zip.ZipException -> ReaderError.CorruptOrInvalid(cause = t)
            is java.io.IOException -> ReaderError.Io(cause = t)
            else -> ReaderError.Internal(cause = t)
        }
}
```

## C8) RenderConfig -> EpubPreferences 映射（支持字体/行高/边距/连字/断词）

**File:** `engines/epub/src/main/kotlin/com/example/reader/engines/epub/internal/render/EpubPreferencesMapper.kt`

```kotlin
package com.example.reader.engines.epub.internal.render

import com.example.reader.api.render.RenderConfig
import org.readium.r2.navigator.epub.EpubPreferences
import org.readium.r2.navigator.epub.FontFamily

internal fun RenderConfig.toEpubPreferences(): EpubPreferences =
    when (this) {
        is RenderConfig.ReflowText -> {
            // Readium 的 fontSize 是倍数（示例：2.0 = 200%）
            val baseSp = 16f
            val fontScale = (fontSizeSp / baseSp).toDouble().coerceIn(0.5, 4.0)

            EpubPreferences(
                fontSize = fontScale,
                lineHeight = lineHeightMult.toDouble().coerceIn(1.0, 3.0),
                paragraphSpacing = (paragraphSpacingDp / 16f).toDouble().coerceIn(0.0, 4.0),
                pageMargins = (pagePaddingDp / 16f).toDouble().coerceIn(0.0, 6.0),
                hyphens = hyphenation,
                fontFamily = fontFamilyName?.toReadiumFontFamily()
            )
        }

        is RenderConfig.FixedPage -> {
            // FXL EPUB 更类似 PDF/图片，字体等偏好通常无效；这里返回空偏好即可
            EpubPreferences()
        }
    }

private fun String.toReadiumFontFamily(): FontFamily? {
    val key = lowercase().trim()
    return when (key) {
        "serif" -> FontFamily.SERIF
        "sans-serif", "sans" -> FontFamily.SANS_SERIF
        "monospace", "mono" -> FontFamily.MONOSPACE
        "cursive" -> FontFamily.CURSIVE
        else -> null
    }
}
```

## C10) 搜索：EpubSearchProvider.kt（基于 Readium SearchService）

**File:** `engines/epub/src/main/kotlin/com/example/reader/engines/epub/internal/provider/EpubSearchProvider.kt`

```kotlin
package com.example.reader.engines.epub.internal.provider

import com.example.reader.api.provider.SearchHit
import com.example.reader.api.provider.SearchOptions
import com.example.reader.api.provider.SearchProvider
import com.example.reader.engines.epub.internal.locator.toAppLocator
import com.example.reader.model.LocatorRange
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow
import org.readium.r2.shared.publication.Publication
import org.readium.r2.shared.publication.services.search
import org.readium.r2.shared.util.getOrElse

internal class EpubSearchProvider(
    private val publication: Publication
) : SearchProvider {

    override fun search(query: String, options: SearchOptions): Flow<SearchHit> = flow {
        val iterator = publication.search(query) ?: return@flow

        var emitted = 0
        try {
            while (true) {
                val page = iterator.next().getOrElse { _ ->
                    // 搜索服务错误：直接停止
                    return@flow
                } ?: break

                for (rl in page.locators) {
                    val app = rl.toAppLocator()
                    val excerpt = buildString {
                        val text = rl.text
                        val before = text.before ?: ""
                        val highlight = text.highlight ?: ""
                        val after = text.after ?: ""
                        append(before)
                        append(highlight)
                        append(after)
                    }.take(240)

                    emit(
                        SearchHit(
                            range = LocatorRange(start = app, end = app),
                            excerpt = excerpt,
                            sectionTitle = rl.title
                        )
                    )

                    emitted++
                    if (emitted >= options.maxHits) return@flow
                }
            }
        } finally {
            runCatching { iterator.close() }
        }
    }
}
```

# D. feature/reader：Compose 中承载 EPUB Fragment 的通用写法（不依赖 engines）

下面给你一个**完全不依赖 engines** 的 `ReaderSurface` Composable：

* 只依赖 `core/common-android` 的 `FragmentRenderSurface` 接口
* 只面向 `ReaderController` 协议
* EPUB 引擎 bindSurface 后自己把 Readium Fragment 放进去
* TXT/PDF 引擎 bindSurface 返回 Ok(Unit) 即可（什么都不做）

**File:** `feature/reader/src/main/java/.../ui/ReaderSurface.kt`

```kotlin
package com.example.reader.feature.reader.ui

import android.view.View
import android.view.ViewGroup
import androidx.compose.runtime.Composable
import androidx.compose.runtime.DisposableEffect
import androidx.compose.runtime.LaunchedEffect
import androidx.compose.runtime.remember
import androidx.compose.ui.Modifier
import androidx.compose.ui.platform.LocalContext
import androidx.compose.ui.viewinterop.AndroidView
import androidx.fragment.app.FragmentActivity
import androidx.fragment.app.FragmentContainerView
import com.example.reader.android.surface.DefaultFragmentRenderSurface
import com.example.reader.api.render.ReaderController

@Composable
fun ReaderSurface(
    controller: ReaderController,
    modifier: Modifier = Modifier
) {
    val activity = LocalContext.current as FragmentActivity
    val fm = activity.supportFragmentManager
    val containerId = remember { View.generateViewId() }

    val surface = remember(fm, containerId) {
        DefaultFragmentRenderSurface(
            fragmentManager = fm,
            containerViewId = containerId
        )
    }

    AndroidView(
        modifier = modifier,
        factory = { ctx ->
            FragmentContainerView(ctx).apply {
                id = containerId
                layoutParams = ViewGroup.LayoutParams(
                    ViewGroup.LayoutParams.MATCH_PARENT,
                    ViewGroup.LayoutParams.MATCH_PARENT
                )
            }
        }
    )

    LaunchedEffect(controller, surface) {
        controller.bindSurface(surface)
    }

    DisposableEffect(controller, surface) {
        onDispose {
            // 解除绑定即可，真正销毁由 session.close/controller.close 决定
            //（避免 Compose 重组时频繁 remove/attach 导致闪烁）
            controller.unbindSurface()
        }
    }
}
```

# E. 你接下来要做的“专业补齐项”（我已把接口留好）

上面这份代码已经能做到：
✅ 打开 EPUB（Readium Streamer）
✅ 用 Readium Navigator 渲染（真实分页/真实定位）
✅ 翻页/跳转/按进度跳转
✅ TOC（Outline）
✅ 搜索（Readium SearchService）
✅ 用户偏好（字体/字号/行高/边距/断词）

要做到你要的“专业读书器”，建议下一步把这 3 件补上：

1. **标注/高亮（Decoration）完整落地**

* `EpubNavigatorFragment` 实现 `DecorableNavigator`，你可以把 `AnnotationRepo` 注入 `EpubSession`，把 `Annotation` 映射成 Readium 的 `Decoration` 并 `applyDecorations()`。
* 同时，把用户选区（SelectableNavigator）回传给 feature，完成“选中→创建 Draft→存库→刷新 decoration”。

2. **TextProvider（复制/分享/导出）**

* 最稳的是接 Readium 的 `ContentService`（`publication.content(start)`），不要自己解析 HTML。
* 你现在 `capabilities.textExtraction=false` 是“专业做法”：宁可不宣称，也不要假支持。

3. **FXL EPUB 能力精细化**

* 通过 `publication.metadata` 的 presentation/layout 判断是否 fixed-layout。
* fixed-layout 时将 `capabilities.fixedLayout=true`、禁用字体偏好（Readium 也说明 FXL 字体不可改），并让 UI 走类似 PDF 的交互策略。
