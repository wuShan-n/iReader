下面给你一套**“可落地的专业版 EPUB 引擎实现”**（基于 Readium Kotlin Toolkit 3.x 的 Streamer + Navigator），完全贴合你当前的模块化约束：

* `engines/epub` **只实现** `core/reader/api` 的契约（ReaderEngine/ReaderDocument/ReaderSession/ReaderController/Provider）
* `feature/*` **不直接依赖** Readium，也不直接依赖 `engines/*`
* UI 侧通过 **core/api 定义的“Android Fragment 承载能力”**（一个很薄的接口）来承载 EPUB Navigator（Readium 的 EpubNavigatorFragment）

> 说明：Readium 官方给出的推荐集成方式是：
>
> * 用 `AssetRetriever + PublicationOpener` 打开 Publication ([GitHub][1])
> * 用 `EpubNavigatorFactory.createFragmentFactory(...)` 创建 `FragmentFactory` 来生成 `EpubNavigatorFragment` ([readium.org][2])
> * 用 `submitPreferences` 动态应用阅读偏好（字体/背景/字号等）([readium.org][3])
> * `OverflowableNavigator.goForward/goBackward` 翻页

## 1) 核心：为 EPUB 引擎提供“UI 承载协议”

因为 Readium 的 EPUB Navigator 是 `Fragment`（EpubNavigatorFragment），而你又不希望 feature 依赖 Readium，所以最干净的方式是：

* 在 `core/reader/api` 里加一个很薄的协议：**“这个 session/controller 是否提供 Android Fragment 作为渲染 surface”**
* feature 只依赖这个协议（AndroidX Fragment 是通用依赖，不算引擎泄漏）

把下面文件放到：`core/reader/api/.../render/AndroidFragmentRenderSurface.kt`

```kotlin
package com.example.reader.contract.render

import androidx.fragment.app.Fragment

/**
 * 可选能力：如果某个引擎的渲染依赖 Fragment（例如 Readium 的 EPUB Navigator），
 * 则 Session/Controller 可以实现这个接口，让上层 UI 承载它。
 *
 * feature 不需要知道 Fragment 的具体类型（也不需要依赖 Readium）。
 */
interface AndroidFragmentRenderSurface {
    /** 引擎提供的渲染 Fragment。UI 层将其 add 到自己的容器中。 */
    fun renderFragment(): Fragment
}
```

> 这一个接口就能把 Readium 的 Fragment “装”进你的 engine，而不让 feature 认识 Readium 类型。

## 4) Readium 组件装配：ReadiumComponents.kt

`engines/epub/.../internal/open/ReadiumComponents.kt`

```kotlin
package com.example.reader.engines.epub.internal.open

import android.content.Context
import org.readium.r2.shared.util.http.DefaultHttpClient
import org.readium.r2.shared.util.asset.AssetRetriever
import org.readium.r2.streamer.PublicationOpener
import org.readium.r2.streamer.parser.DefaultPublicationParser

/**
 * 把 Readium 的核心对象集中管理，避免到处 new。
 *
 * - AssetRetriever：把 File/Uri/Url 变成 Asset
 * - PublicationOpener：把 Asset 解析成 Publication
 */
internal class ReadiumComponents(
    context: Context,
) {

    private val httpClient = DefaultHttpClient()

    // 这里的构造函数签名在不同版本 Readium 可能略有差异；
    // 按官方 guide 的思路：AssetRetriever 依赖 ContentResolver 与 HttpClient。
    private val assetRetriever: AssetRetriever =
        AssetRetriever(context.contentResolver, httpClient)

    private val parser = DefaultPublicationParser(
        context = context,
        httpClient = httpClient,
        assetRetriever = assetRetriever,
        pdfFactory = null, // EPUB 引擎不需要 PDF
    )

    val publicationOpener: PublicationOpener =
        PublicationOpener(
            parser = parser,
            contentProtections = emptyList(), // 若支持 LCP，可在这里注入
        )

    suspend fun retrieveAsset(url: org.readium.r2.shared.util.AbsoluteUrl) =
        assetRetriever.retrieve(url)

    suspend fun retrieveAsset(file: java.io.File) =
        assetRetriever.retrieve(file)
}
```

## 5) 打开 EPUB：EpubOpener.kt

`engines/epub/.../internal/open/EpubOpener.kt`

```kotlin
package com.example.reader.engines.epub.internal.open

import android.content.Context
import com.example.reader.contract.engine.ReaderDocument
import com.example.reader.contract.error.ReaderError
import com.example.reader.contract.error.ReaderResult
import com.example.reader.contract.open.OpenOptions
import com.example.reader.contract.source.DocumentSource
import com.example.reader.engines.epub.internal.doc.EpubDocument
import kotlinx.coroutines.CoroutineDispatcher
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.withContext
import org.readium.r2.shared.util.AbsoluteUrl
import org.readium.r2.shared.util.Try
import org.readium.r2.shared.util.toAbsoluteUrl
import org.readium.r2.shared.util.toUrl
import javax.inject.Inject
import javax.inject.Singleton

@Singleton
internal class EpubOpener @Inject constructor(
    private val appContext: Context,
    private val io: CoroutineDispatcher = Dispatchers.IO,
) {
    private val readium = ReadiumComponents(appContext)

    suspend fun open(
        source: DocumentSource,
        options: OpenOptions,
    ): ReaderResult<ReaderDocument> = withContext(io) {

        val urlOrFile = source.toReadiumLocation(appContext)
            ?: return@withContext ReaderResult.failure(
                ReaderError.Unsupported("Unsupported DocumentSource for EPUB: $source")
            )

        // 1) retrieve Asset
        val assetTry: Try<org.readium.r2.shared.util.asset.Asset, *> = when (urlOrFile) {
            is ReadiumLocation.FileLoc -> readium.retrieveAsset(urlOrFile.file)
            is ReadiumLocation.UrlLoc -> readium.retrieveAsset(urlOrFile.url)
        }

        val asset = assetTry.getOrNull()
            ?: return@withContext ReaderResult.failure(
                ReaderError.OpenFailed("Readium AssetRetriever failed: ${assetTry}")
            )

        // 2) open Publication
        val pubTry = readium.publicationOpener.open(
            asset = asset,
            allowUserInteraction = options.allowUserInteraction,
        )

        val publication = pubTry.getOrNull()
        if (publication == null) {
            // 失败一定记得关 asset
            asset.close()
            return@withContext ReaderResult.failure(
                ReaderError.OpenFailed("Readium PublicationOpener failed: $pubTry")
            )
        }

        // 3) build our ReaderDocument
        val doc = EpubDocument(
            source = source,
            asset = asset,
            publication = publication,
        )

        ReaderResult.success(doc)
    }
}

/**
 * 统一把 DocumentSource 变成 Readium 可接受的 location：
 * - File -> File
 * - Uri  -> AbsoluteUrl(content://...)
 */
internal sealed class ReadiumLocation {
    data class FileLoc(val file: java.io.File) : ReadiumLocation()
    data class UrlLoc(val url: AbsoluteUrl) : ReadiumLocation()
}

/**
 * 你实际的 DocumentSource 结构可能不同，按你的 core/files 去改这里的 when 分支即可。
 */
internal fun DocumentSource.toReadiumLocation(context: Context): ReadiumLocation? {
    return when (this) {
        is DocumentSource.File -> ReadiumLocation.FileLoc(file)
        is DocumentSource.Uri -> {
            val abs = uri.toAbsoluteUrl()
            abs?.let { ReadiumLocation.UrlLoc(it) }
        }
        // 若你支持 http(s) 远程：
        is DocumentSource.HttpUrl -> AbsoluteUrl(url)?.let { ReadiumLocation.UrlLoc(it) }

        else -> null
    }
}

/**
 * 给 Try 提供一个 getOrNull（Readium Try 是自定义的）。
 * 如果你项目里已有类似扩展，可删掉。
 */
internal fun <S, F> Try<S, F>.getOrNull(): S? =
    when (this) {
        is Try.Success -> this.value
        is Try.Failure -> null
    }
```

## 6) Document：EpubDocument.kt

`engines/epub/.../internal/doc/EpubDocument.kt`

```kotlin
package com.example.reader.engines.epub.internal.doc

import com.example.reader.contract.engine.ReaderDocument
import com.example.reader.contract.engine.ReaderSession
import com.example.reader.contract.model.BookFormat
import com.example.reader.contract.model.DocumentCapabilities
import com.example.reader.contract.model.DocumentId
import com.example.reader.contract.model.DocumentMetadata
import com.example.reader.contract.open.OpenOptions
import com.example.reader.contract.render.RenderConfig
import com.example.reader.contract.source.DocumentSource
import com.example.reader.engines.epub.internal.provider.EpubOutlineProvider
import com.example.reader.engines.epub.internal.provider.EpubResourceProvider
import com.example.reader.engines.epub.internal.provider.EpubSearchProvider
import com.example.reader.engines.epub.internal.session.EpubSession
import org.readium.r2.shared.publication.Publication
import org.readium.r2.shared.util.asset.Asset
import java.security.MessageDigest

internal class EpubDocument(
    private val source: DocumentSource,
    private val asset: Asset,
    internal val publication: Publication,
) : ReaderDocument {

    override val format: BookFormat = BookFormat.EPUB

    override val id: DocumentId = DocumentId(hashStableId(source))

    override val metadata: DocumentMetadata = publication.toDocumentMetadata()

    override val capabilities: DocumentCapabilities = DocumentCapabilities(
        // 你可以在这里基于 publication.services/metadata 判断能力
        supportsOutline = publication.tableOfContents.isNotEmpty(),
        supportsSearch = publication.isSearchable(),
        supportsReflow = true,
    )

    override val outlineProvider = EpubOutlineProvider(publication)

    override val resourceProvider = EpubResourceProvider(publication)

    override val searchProvider = EpubSearchProvider(publication)

    override suspend fun openSession(
        initial: com.example.reader.contract.model.Locator?,
        renderConfig: RenderConfig,
        options: OpenOptions,
    ): ReaderSession {
        return EpubSession(
            documentId = id,
            publication = publication,
            initialLocator = initial,
            initialRenderConfig = renderConfig,
        )
    }

    override fun close() {
        // 关闭顺序：publication -> asset
        try { publication.close() } catch (_: Throwable) {}
        try { asset.close() } catch (_: Throwable) {}
    }
}

private fun hashStableId(source: DocumentSource): String {
    val raw = source.toString()
    val md = MessageDigest.getInstance("SHA-256")
    val bytes = md.digest(raw.toByteArray())
    return bytes.joinToString("") { "%02x".format(it) }
}

// ======== Publication -> core metadata/capability ========

private fun Publication.toDocumentMetadata(): DocumentMetadata {
    val md = manifest.metadata
    return DocumentMetadata(
        title = md.localizedTitle?.string() ?: "Untitled",
        authors = md.authors.mapNotNull { it.localizedName.stringOrNull() },
        identifier = md.identifier,
        languages = md.languages,
        description = md.description,
    )
}

private fun org.readium.r2.shared.publication.LocalizedString.string(): String =
    translations[null]?.string ?: translations.values.firstOrNull()?.string ?: ""

private fun org.readium.r2.shared.publication.LocalizedString.stringOrNull(): String? =
    translations[null]?.string ?: translations.values.firstOrNull()?.string

private fun Publication.isSearchable(): Boolean {
    // Publication.search(...) 会返回 null 表示不可搜索，这里粗略判定一下
    // 更严谨：你可以 findServices(SearchService::class) 判断
    return true
}
```

> 上面 `DocumentMetadata/DocumentCapabilities/ReaderDocument` 的字段你需要按你 core/model 实际定义对齐。
> 核心思路是：**EpubDocument 持有 Publication，并把“能力”变成 core 里的 Provider**。

### 7.2 EpubController.kt

`engines/epub/.../internal/session/EpubController.kt`

```kotlin
package com.example.reader.engines.epub.internal.session

import androidx.fragment.app.Fragment
import com.example.reader.contract.engine.ReaderController
import com.example.reader.contract.model.DocumentId
import com.example.reader.contract.model.Locator as CoreLocator
import com.example.reader.contract.render.RenderConfig
import com.example.reader.engines.epub.internal.locator.EpubLocatorMapper
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.SupervisorJob
import kotlinx.coroutines.flow.SharingStarted
import kotlinx.coroutines.flow.StateFlow
import kotlinx.coroutines.flow.map
import kotlinx.coroutines.flow.stateIn
import kotlinx.coroutines.launch
import org.readium.r2.navigator.epub.EpubNavigatorFactory
import org.readium.r2.navigator.epub.EpubNavigatorFragment
import org.readium.r2.navigator.epub.EpubPreferences
import org.readium.r2.shared.publication.Publication

/**
 * ✅ 专业版关键点：Controller 里持有 Readium 的 EpubNavigatorFragment，
 * 但对外只暴露 core/api 的 ReaderController 能力。
 */
internal class EpubController(
    private val documentId: DocumentId,
    private val publication: Publication,
    initialLocator: CoreLocator?,
    initialRenderConfig: RenderConfig,
) : ReaderController {

    private val scope = CoroutineScope(SupervisorJob() + Dispatchers.Main.immediate)

    private val locatorMapper = EpubLocatorMapper(publication)

    private val navigatorFactory = EpubNavigatorFactory(publication)

    private val fragmentFactory = navigatorFactory.createFragmentFactory(
        initialLocator = initialLocator?.let(locatorMapper::toReadium),
        initialPreferences = initialRenderConfig.toEpubPreferences(),
        listener = object : EpubNavigatorFragment.Listener {},
        paginationListener = null,
    )

    private val fragment: EpubNavigatorFragment =
        fragmentFactory.instantiate(
            EpubNavigatorFragment::class.java.classLoader!!,
            EpubNavigatorFragment::class.java.name
        ) as EpubNavigatorFragment

    // 把 Readium 的 currentLocator 映射成你 core/model 的 Locator
    val currentLocator: StateFlow<CoreLocator?> =
        fragment.currentLocator
            .map { locatorMapper.toCore(it) }
            .stateIn(scope, SharingStarted.Eagerly, initialLocator)

    fun renderFragment(): Fragment = fragment

    // ================== ReaderController API ==================

    override fun goTo(locator: CoreLocator, animated: Boolean): Boolean {
        return fragment.go(locatorMapper.toReadium(locator), animated)
    }

    override fun nextPage(): Boolean = fragment.goForward()

    override fun prevPage(): Boolean = fragment.goBackward()

    override fun applyRenderConfig(config: RenderConfig) {
        val prefs = config.toEpubPreferences()
        scope.launch {
            fragment.submitPreferences(prefs)
        }
    }

    override fun close() {
        scope.cancel()
        // Fragment 的销毁由宿主 FragmentManager 管理
    }
}

/**
 * RenderConfig -> Readium EpubPreferences
 * Readium 支持非常多的偏好项（字号/字体/背景/分栏/行距/对齐/边距...）:contentReference[oaicite:6]{index=6}
 */
private fun RenderConfig.toEpubPreferences(): EpubPreferences {
    // ⚠️ 你按自己 core/datastore 的设置字段对齐即可，这里给出常用映射骨架
    return EpubPreferences(
        fontSize = fontSize,                 // e.g. Double? or Float?（按你的 RenderConfig）
        fontFamily = fontFamily,             // String?
        textColor = textColor,               // Readium Color type
        backgroundColor = backgroundColor,   // Readium Color type
        columnCount = if (twoPageSpread) 2 else 1
    )
}
```

> 你需要把 `ReaderController` 的方法名对齐（比如你可能叫 `go(locator)` / `goNext()` / `setRenderConfig()`）。
> 关键要点是：**Controller 内部完全掌控 Readium Fragment**，对外暴露的是你 core/api 的抽象能力。

## 8) Locator 映射：EpubLocatorMapper.kt

`engines/epub/.../internal/locator/EpubLocatorMapper.kt`

```kotlin
package com.example.reader.engines.epub.internal.locator

import com.example.reader.contract.model.Locator as CoreLocator
import org.readium.r2.shared.publication.Publication
import org.readium.r2.shared.publication.Locator as ReadiumLocator
import org.readium.r2.shared.util.Url
import org.readium.r2.shared.util.toUrl

/**
 * 核心：把 Readium Locator <-> 你的 core/model/Locator 映射。
 *
 * 最推荐做法：你的 core Locator 直接兼容 Readium Locator 的结构（href + locations.fragments/progression/position/totalProgression + text）
 * 这样搜索/书签/进度存储都会很稳定。
 */
internal class EpubLocatorMapper(
    private val publication: Publication,
) {
    fun toReadium(core: CoreLocator): ReadiumLocator {
        return ReadiumLocator(
            href = Url(core.href),
            mediaType = org.readium.r2.shared.util.mediatype.MediaType(core.mediaType),
            title = core.title,
            locations = ReadiumLocator.Locations(
                fragments = core.locations.fragments,
                progression = core.locations.progression,
                position = core.locations.position,
                totalProgression = core.locations.totalProgression,
                otherLocations = core.locations.otherLocations
            ),
            text = ReadiumLocator.Text(
                before = core.text.before,
                highlight = core.text.highlight,
                after = core.text.after
            )
        )
    }

    fun toCore(readium: ReadiumLocator): CoreLocator {
        return CoreLocator(
            href = readium.href.toString(),
            mediaType = readium.mediaType.toString(),
            title = readium.title,
            locations = CoreLocator.Locations(
                fragments = readium.locations.fragments,
                progression = readium.locations.progression,
                position = readium.locations.position,
                totalProgression = readium.locations.totalProgression,
                otherLocations = readium.locations.otherLocations
            ),
            text = CoreLocator.Text(
                before = readium.text.before,
                highlight = readium.text.highlight,
                after = readium.text.after
            )
        )
    }
}
```

> 你 core/model 的 `Locator` 结构如果跟 Readium 很接近，你会省掉后续 80% 的坑。

## 9) Provider 实现

### 9.1 OutlineProvider：EpubOutlineProvider.kt

Readium 提供 `Publication.tableOfContents`（links tree），也提供 `locatorFromLink(link)` 可把 TOC link 转成 Locator ([readium.org][6])

`engines/epub/.../internal/provider/EpubOutlineProvider.kt`

```kotlin
package com.example.reader.engines.epub.internal.provider

import com.example.reader.contract.model.Outline
import com.example.reader.contract.provider.OutlineProvider
import com.example.reader.engines.epub.internal.locator.EpubLocatorMapper
import org.readium.r2.shared.publication.Publication
import org.readium.r2.shared.publication.Link

internal class EpubOutlineProvider(
    private val publication: Publication,
) : OutlineProvider {

    private val mapper = EpubLocatorMapper(publication)

    override suspend fun getOutline(): Outline {
        return Outline(
            nodes = publication.tableOfContents.mapNotNull { it.toOutlineNode() }
        )
    }

    private fun Link.toOutlineNode(): Outline.Node? {
        val title = title ?: return null
        val locator = publication.locatorFromLink(this)?.let(mapper::toCore)

        return Outline.Node(
            title = title,
            locator = locator,
            children = children.mapNotNull { it.toOutlineNode() }
        )
    }
}
```

`engines/epub/.../internal/provider/EpubResourceProvider.kt`

```kotlin
package com.example.reader.engines.epub.internal.provider

import com.example.reader.contract.provider.ResourceProvider
import org.readium.r2.shared.publication.Publication
import org.readium.r2.shared.publication.Link
import org.readium.r2.shared.util.Url

internal class EpubResourceProvider(
    private val publication: Publication,
) : ResourceProvider {

    override suspend fun readBytes(href: String): ByteArray? {
        val res = publication.get(Url(href)) ?: return null
        return res.read().getOrNull()
    }

    override suspend fun readBytes(link: com.example.reader.contract.model.Link): ByteArray? {
        val readiumLink = Link(href = org.readium.r2.shared.publication.Href(link.href))
        val res = publication.get(readiumLink) ?: return null
        return res.read().getOrNull()
    }
}
```

### 9.3 SearchProvider：EpubSearchProvider.kt

Readium 的 `Publication.search(query)` 会返回 `SearchIterator?`，不可搜索时返回 null ([readium.org][8])
Iterator 可以 `next()` 翻页式取结果 ([readium.org][9])

`engines/epub/.../internal/provider/EpubSearchProvider.kt`

```kotlin
package com.example.reader.engines.epub.internal.provider

import com.example.reader.contract.error.ReaderError
import com.example.reader.contract.error.ReaderResult
import com.example.reader.contract.provider.SearchProvider
import com.example.reader.engines.epub.internal.locator.EpubLocatorMapper
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.withContext
import org.readium.r2.shared.publication.Publication
import org.readium.r2.shared.publication.services.search.search

internal class EpubSearchProvider(
    private val publication: Publication,
) : SearchProvider {

    private val mapper = EpubLocatorMapper(publication)

    override suspend fun search(query: String): ReaderResult<List<com.example.reader.contract.model.Locator>> =
        withContext(Dispatchers.IO) {
            val iterator = publication.search(query)
                ?: return@withContext ReaderResult.failure(
                    ReaderError.Unsupported("Publication is not searchable")
                )

            val out = mutableListOf<com.example.reader.contract.model.Locator>()

            try {
                while (true) {
                    val pageTry = iterator.next()
                    val page = pageTry.getOrNull() ?: break
                    val locators = page.locators.map(mapper::toCore)
                    out += locators
                }
            } finally {
                iterator.close()
            }

            ReaderResult.success(out)
        }
}
```

## 11) feature/reader 侧如何承载 EPUB（不依赖 Readium）

你的 `feature/reader` 只知道：

* `session.controller`
* controller 是否具备 `AndroidFragmentRenderSurface`（core/api 类型）
* 把 `renderFragment()` add 到容器即可

伪代码（Compose 宿主 FragmentContainerView 的方式）：

```kotlin
@Composable
fun ReaderHost(session: ReaderSession) {
    val surface = session as? AndroidFragmentRenderSurface
    if (surface != null) {
        AndroidView(factory = { context ->
            FragmentContainerView(context).apply { id = View.generateViewId() }
        }, update = { container ->
            val fragment = surface.renderFragment()
            (context as FragmentActivity).supportFragmentManager
                .beginTransaction()
                .replace(container.id, fragment)
                .commitNowAllowingStateLoss()
        })
    } else {
        // 其他引擎的渲染方式（TXT 自绘 / PDF tiles 等）
    }
}
```

# 这套实现的关键优势（为什么我推荐它）

1. **解析/定位/搜索/目录/渲染全部交给 Readium**：少走 1~2 年坑（尤其是 EPUB CFI、spine、内嵌资源、排版、跨章节定位等）。([GitHub][10])
2. **feature 永远不依赖 Readium**：你只在 engine 模块依赖三方库，符合你“引擎可替换”的模块约束。
3. **可演进**：以后你要做

   * 离线全文索引（WorkManager + ContentService）
   * 高亮/笔记（DecorableNavigator）
   * TTS（Readium media navigator）

   都能在 `engines/epub/internal/*` 内继续扩，不污染 feature。
