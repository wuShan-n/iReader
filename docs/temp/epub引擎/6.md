## 新增 1：分页签名（稳定 key，忽略 config.extra，避免循环）

### `engines/epub/.../internal/pagination/PaginationSignature.kt`（新增）

```kotlin
package com.example.reader.engines.epub.internal.pagination

import com.example.reader.api.render.LayoutConstraints
import com.example.reader.api.render.RenderConfig
import kotlin.math.roundToInt

internal object PaginationSignature {

    /**
     * 只使用影响分页的参数（不包含 config.extra），确保：
     * - feature 往 extra 塞 metrics 不会导致 sig 改变 -> 不会无限循环
     */
    fun of(config: RenderConfig.ReflowText, constraints: LayoutConstraints?): Int {
        val vw = constraints?.viewportWidthPx ?: 0
        val vh = constraints?.viewportHeightPx ?: 0
        val d = ((constraints?.density ?: 1f) * 100).roundToInt()
        val fs = ((constraints?.fontScale ?: 1f) * 100).roundToInt()

        fun f(x: Float) = (x * 100).roundToInt()

        var h = 17
        h = 31 * h + vw
        h = 31 * h + vh
        h = 31 * h + d
        h = 31 * h + fs
        h = 31 * h + f(config.fontSizeSp)
        h = 31 * h + f(config.lineHeightMult)
        h = 31 * h + f(config.paragraphSpacingDp)
        h = 31 * h + f(config.pagePaddingDp)
        h = 31 * h + (config.fontFamilyName?.hashCode() ?: 0)
        h = 31 * h + if (config.hyphenation) 1 else 0
        return h
    }
}
```

## 修改 2：HtmlComposer 上报 metrics（真实页数/当前页）+ 带 spine+sig

### `engines/epub/.../internal/render/HtmlComposer.kt`（修改）

> 改点：
>
> * JS 在 `load` 后计算：`pages = ceil(scrollWidth / VW)`、`page = round(scrollLeft / VW)`
> * 通过 `reader://metrics?spine=...&sig=...&pages=...&page=...` 回传给 feature
> * scroll 变化也会 debounce 上报（用户手动滑动/anchor跳转会更准）

1. 新增参数：把 `compose(...)` 签名加两个参数（只改签名与调用处）

```kotlin

spineIndexForMetrics: Int,
```

2. 模板里新增两个占位符并替换：
   在 JS 常量区加：

```js
const SPINE = __SPINE__;
const SIG = __SIG__;
```

3. 新增 reportMetrics() + scroll debounce（把下面这段插进你原 JS IIFE 里即可）

```js
let __t = 0;
function reportMetrics(){
  if(!VW || VW <= 0) return;
  const scroller = document.scrollingElement || document.documentElement;
  const sw = scroller.scrollWidth || 0;
  const pages = Math.max(1, Math.ceil(sw / VW));
  const page = Math.max(0, Math.round((scroller.scrollLeft || 0) / VW));
  window.location.href = 'reader://metrics?spine=' + SPINE + '&sig=' + SIG + '&pages=' + pages + '&page=' + page;
}

function debounceReport(){
  if(__t) clearTimeout(__t);
  __t = setTimeout(reportMetrics, 120);
}

window.addEventListener('scroll', function(){
  debounceReport();
}, {passive:true});
```

4. 在 `load` 事件末尾加一次上报（保证首次也有）

```js
window.addEventListener('load', function(){
  goToPage();
  goToAnchor();
  setTimeout(reportMetrics, 0);
});
```

5. 返回时替换占位符（在最终返回 tpl.replace(...) 链上加两行）

```kotlin
.replace("__SPINE__", spineIndexForMetrics.toString())
.replace("__SIG__", sig.toString())
```

## 新增 2：Feature 侧 metrics 路由（把真实页数写进 config.extra，并对齐当前页）

> 关键点：feature **不依赖 engines**，只用 core api：
>
> * 从 `reader://metrics?...` 取 `spine/sig/pages/page`
> * 把真实页数写入 `RenderConfig.ReflowText.extra`：`epub.pageCount.<spine>.<sig>`
> * 如果 WebView 报告的当前页 ≠ controller.state 当前页，则 `goTo(reflow.page)` 对齐（避免 state 漂移）

### `feature/reader/.../web/ReaderWebViewLinkRouter.kt`（修改：新增 metrics 分支 + 去重）

在现有 `tryHandle()` 里增加一个分支，并加一点去重缓存：

```kotlin
// 新增 import
import java.util.concurrent.ConcurrentHashMap
import com.example.reader.api.render.RenderConfig
import com.example.reader.model.LocatorSchemes

object ReaderWebViewLinkRouter {

    private val lastPages = ConcurrentHashMap<String, Int>() // key=spine:sig -> pages
    private val lastPage = ConcurrentHashMap<String, Int>()  // key=spine:sig -> page

    fun tryHandle(url: String, controller: ReaderController, scope: CoroutineScope): Boolean {
        val uri = runCatching { Uri.parse(url) }.getOrNull() ?: return false
        if (uri.scheme != "reader") return false

        return when (uri.host) {

            "metrics" -> {
                val spine = uri.getQueryParameter("spine")?.toIntOrNull() ?: return true
                val sig = uri.getQueryParameter("sig")?.toIntOrNull() ?: return true
                val pages = uri.getQueryParameter("pages")?.toIntOrNull() ?: return true
                val page = uri.getQueryParameter("page")?.toIntOrNull() ?: return true

                val key = "$spine:$sig"

                scope.launch {
                    // 1) 写入真实页数到 config.extra（仅当变化）
                    val cfg = controller.state.value.config
                    val reflow = cfg as? RenderConfig.ReflowText
                    if (reflow != null) {
                        val extraKey = "epub.pageCount.$spine.$sig"
                        val old = reflow.extra[extraKey]?.toIntOrNull()
                        val prev = lastPages[key]
                        if (old != pages && prev != pages) {
                            lastPages[key] = pages
                            val newCfg = reflow.copy(extra = reflow.extra + (extraKey to pages.toString()))
                            controller.setConfig(newCfg)
                        }
                    }

                    // 2) 对齐当前页（仅当变化）
                    val prevPage = lastPage[key]
                    val currentLoc = controller.state.value.locator
                    val expectedValue = "$spine:$page:$sig"
                    val already = currentLoc.scheme == LocatorSchemes.REFLOW_PAGE && currentLoc.value.startsWith("$spine:$page")
                    if (!already && prevPage != page) {
                        lastPage[key] = page
                        controller.goTo(Locator(LocatorSchemes.REFLOW_PAGE, expectedValue))
                    }
                }
                true
            }

            // 原有 goto / external 分支保持不变
            else -> { /* 你原来的逻辑 */ true }
        }
    }
}
```

> 注意：这里对齐页用的是 `startsWith("$spine:$page")`，兼容你旧的 2 段/3 段 value。

# 现在新增实现了什么（这轮交付）

* ✅ **WebView 实测真实页数**：每章真正 columns 计算后的 pages 回传到 feature
* ✅ **分页从估算升级到真实**：controller 的 `currentPageCount()` 通过 `config.extra` 覆盖后变成真实页数
* ✅ **state 与实际页面对齐**：WebView 报告当前页，feature 触发 `goTo(reflow.page)`，避免 state 漂移
* ✅ **anchor 跳转更准**：`#frag` 用“offset/len * pageCount”映射页数（且 pageCount 已可真实）
