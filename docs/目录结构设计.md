│   ├── library/                        # 书架、导入、管理
│   ├── reader/                         # 阅读页统一入口（根据格式选择引擎）
│   ├── annotations/                    # 标注/书签/笔记
│   ├── search/                         # 全书/全文搜索（可选）
│   └── settings/                       # 设置 UI + usecase

├── benchmark/                          # 可选：Macrobenchmark（翻页、打开书、搜索性能）

## 3) 推荐的依赖规则（避免“模块互相引用成一团”）

建议在 `docs/architecture.md` 固化以下规则（并可用 build-logic 写一个简单校验）：

- `feature/*` **只能依赖**：

    - `core/*`（designsystem、navigation、data、datastore、reader/api 或 reader/runtime）

    - **不能直接依赖 `engines/*`**

- `engines/*` **只能依赖**：

    - `core/reader/api`

    - `core/common` / `core/model` / `core/files`（少量必要的基础设施）

- `core/*` **不能依赖 feature**

- `app` 才是“组装者”：

    - 依赖所有 feature + engines

    - 在 DI 中把 engines 注册到 `EngineRegistry`


这样 feature 永远不会因为引擎替换而被迫改动。

## 4) feature 模块内部结构（更落地的一版）

以 `feature/reader` 为例（其他 feature 类似）：

```text
feature/reader/src/main/java/.../reader/
├── navigation/
│   ├── ReaderRoute.kt                  # route/args 定义（可配合 core/navigation）
│   └── ReaderNavGraph.kt               # 只做图声明，不写业务
├── ui/
│   ├── ReaderScreen.kt                 # 纯 UI（Compose）
│   ├── ReaderScaffold.kt               # 工具栏/进度条/菜单抽成壳
│   └── components/                     # 可复用组件
├── presentation/
│   ├── ReaderViewModel.kt
│   ├── ReaderUiState.kt
│   └── ReaderIntent.kt                 # MVI 可选
├── domain/
│   ├── usecase/
│   │   ├── OpenBook.kt
│   │   ├── ObserveReaderSettings.kt
│   │   ├── SaveProgress.kt
│   │   └── ToggleBookmark.kt
│   └── model/                          # 仅该 feature 私有的模型（若需要）
├── di/
│   └── ReaderModule.kt                 # Hilt/Koin：绑定 VM、usecase
└── mapper/                             # UI <-> domain 的映射（可选）
```

> 注意：`Repository` 接口如果是跨 feature 共享（Book/Progress/Annotation），建议放 `core/data`；  
> feature 里只保留“组合 usecase 与 UI 状态”的逻辑。

## 5) engines 模块内部结构（把“公共入口”与“内部实现”分开）

以 `engines/epub` 为例：

```text
engines/epub/src/main/java/.../epub/
├── EpubEngine.kt                       # ✅ 对外唯一入口：实现 ReaderEngine，注册 capabilities
├── di/
│   └── EpubEngineModule.kt             # 可选：引擎内部 DI（或由 app 统一装配）
└── internal/                           # ✅ 全部实现细节放 internal（配合 Kotlin internal）
    ├── open/
    │   ├── EpubOpener.kt               # 打开/校验/解压/缓存策略
    │   └── EpubDocument.kt             # 实现 ReaderDocument
    ├── parser/
    │   ├── EpubParser.kt
    │   └── model/
    ├── locator/
    │   ├── EpubCfi.kt
    │   └── LocatorMapper.kt
    ├── render/
    │   ├── EpubRenderer.kt             # WebView/ComposeText/自研排版
    │   ├── StyleResolver.kt
    │   └── LayoutEngine.kt
    ├── provider/
    │   ├── EpubOutlineProvider.kt
    │   ├── EpubTextProvider.kt
    │   └── EpubSearchProvider.kt
    └── cache/
        ├── ResourceCache.kt
        └── UnzipManager.kt
```

同样地，`engine-pdf` 可把 `text extraction/search` 做成可插拔（开关能力）：

```text

├── PdfEngine.kt
└── internal/
    ├── render/                         # PdfRenderer / Pdfium / third-party
    ├── outline/
    ├── text/                           # 可选：文本抽取
    ├── search/                         # 可选：全文搜索
    └── cache/
```

## 6) core/reader/api


一个更清晰的 api 目录可以这样：

```text
core/reader/api/
└── src/main/java/.../reader/api/
    ├── engine/                         # ReaderEngine / ReaderSession / ReaderDocument
    ├── model/                          # BookFormat / Locator / Outline / Metadata / Capabilities
    ├── open/                           # OpenOptions / Source
    ├── render/                         # RenderConfig / Page / Controller / TextMapping / Tiles
    ├── provider/                       # Outline/Search/Text/Resource/Annotation
    ├── annotation/                     # Annotation / Decoration
    └── error/                          # ReaderError / ReaderResult
```

然后 `core/reader/runtime` 放：

```text
core/reader/runtime/
├── registry/
│   ├── EngineRegistryImpl.kt
│   └── EngineBinder.kt                 # app 通过多绑定把引擎塞进来
├── session/
│   ├── ReaderSessionManager.kt
│   └── SessionStore.kt                 # 进程重建恢复可选
├── policy/
│   ├── DefaultRenderPolicy.kt
│   ├── CachePolicy.kt
│   └── PrefetchPolicy.kt
└── di/
    └── ReaderRuntimeModule.kt
```
